현재 작업 폴더의 내용물을 상세히 탐색하고 요약합니다.

## 절차

1. **프로젝트 루트 구조 파악**: 현재 작업 디렉토리의 최상위 파일과 폴더를 나열합니다.
2. **핵심 설정 파일 확인**: CLAUDE.md, package.json, pyproject.toml, Makefile, docker-compose.yml 등 프로젝트 설정 파일이 있으면 읽고 요약합니다.
3. **Git 상태 확인**: 현재 브랜치, 최근 커밋 5개, 변경 사항(staged/unstaged/untracked)을 확인합니다.
4. **주요 디렉토리 탐색**: 각 하위 디렉토리의 목적과 포함된 파일 수를 파악합니다. 깊이 2단계까지 탐색합니다.
5. **파일 유형 통계**: 확장자별 파일 개수를 집계합니다.
6. **NEXT_SESSION.md 확인**: 파일이 있으면 읽어서 이전 세션의 맥락을 파악합니다.
7. **Memory 확인**: .claude/memory/ 디렉토리에 저장된 메모리가 있으면 확인합니다.

## 출력 형식

아래 형식으로 요약을 출력합니다:

```
## 프로젝트 개요
- 프로젝트 이름/목적 (설정 파일에서 추론)
- 주요 기술 스택

## 폴더 구조
(트리 형태로 주요 디렉토리와 역할 설명)

## 파일 통계
- 총 파일 수
- 주요 확장자별 분포

## Git 상태
- 현재 브랜치
- 최근 커밋 요약
- 작업 중인 변경 사항

## 이전 세션 컨텍스트
(NEXT_SESSION.md가 있는 경우)

## 다음 단계 제안
(발견한 내용을 바탕으로 할 수 있는 작업 제안)
```

## 규칙

- 바이너리 파일이나 .env, node_modules, .git, __pycache__, venv 등 생성된 디렉토리 내부는 탐색하지 않습니다.
- 파일 내용을 읽을 때는 설정/문서 파일 위주로만 읽습니다. 소스 코드는 구조만 파악합니다.
- 출력은 한국어로 합니다.
- 간결하되 놓치는 정보가 없도록 합니다.
